### Gate: find_logical_fallacies

An edit fails this gate if it lifts fallacious reasoning from the source into the wiki as if it were sound. The wiki shouldn't launder bad arguments.

Common patterns to watch for:
- **Appeal to authority** as sole support — "X said it, so it's true" — needs the reasoning X gave, not just the attribution.
- **Hasty generalization** — a single example treated as a universal rule.
- **False dichotomy** — "either X or Y" when a third option is obvious.
- **Non sequitur** — the conclusion doesn't follow from the premise the source states.
- **Strawman summary** — capturing an opponent's view in a way the opponent wouldn't recognize.

Remediation:
1. **Tighten** — attribute the fallacy to the source's framing rather than present it as fact. "The author frames it as X" rather than "it is X".
2. Add a parenthetical qualifier noting the reasoning move if it's load-bearing.
3. **Drop** only if the entire edit hinges on the fallacy and can't be rescued.
