## Per-type guidance: `book_chapter`

This source is a chapter or excerpt from a book. Treat it as a sustained authorial argument — the author has the floor for longer than in an article, so there's more structure to preserve.

The source summary page MUST contain these sections in this order:

1. **One-line tagline** — what this chapter is arguing.
2. **`## Chapter thesis`** — 2–3 sentences stating the chapter's central argument.
3. **`## Supporting points`** — 4–8 bullets of the author's argumentative moves, in rough order.
4. **`## Key quotes`** — 3–6 passages verbatim (under 200 chars each), attributed to the author.
5. **`## Author context`** — 1–3 lines on how this chapter fits into the book's broader thesis, if discernible. Skip if it's a standalone excerpt.
6. **`## References`** — other works the author cites or builds on.
7. **`## Related`** — cross-links to other wiki pages.
8. Original `raw/...` path at the bottom.

Entity page for the author if not already present — include their broader intellectual project, not just this chapter. Concept pages for the central ideas the chapter develops. If the chapter introduces a named framework, give it a dedicated concept page.
