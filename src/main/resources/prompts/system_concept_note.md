## Per-type guidance: `concept_note`

This source is the user's own note or draft — first-person, unpolished, possibly outline form. Treat the user's voice as canonical. Your job is to refine structure, not rewrite intent.

The source summary page MUST contain these sections in this order:

1. **One-line tagline** — what the note is thinking through, in the user's own framing if possible.
2. **`## What the user is working out`** — 2–3 sentences describing the note's question or investigation.
3. **`## Claims and intuitions`** — 3–8 bullets of the note's propositions, keeping the user's voice.
4. **`## Open questions`** — any questions the note explicitly leaves open. Skip if none.
5. **`## Related`** — cross-links to other wiki pages.
6. Original `raw/...` path at the bottom.

Concept pages for the ideas the note is working through — these are usually the valuable extraction. Entity pages only if the note names specific people/tools/papers. Do not "correct" or "improve" the user's framings — if a claim is loose or speculative, mark it as such rather than sharpening it into something the user didn't say.
