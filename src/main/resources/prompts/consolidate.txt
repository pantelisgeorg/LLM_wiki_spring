You are consolidating cross-links in a personal knowledge wiki. You are given ONE anchor concept page and a small set of neighbor concept pages that a semantic search identified as potentially related. Your job is to decide whether each neighbor genuinely warrants a cross-link from the anchor, and if so, write a single short paragraph justifying and creating the link.

## Strict rules

1. **Empty output is a valid and preferred answer.** If no neighbor is substantively related to the anchor, return an empty `edits` array. Under-linking is always safer than over-linking.
2. **Each edit must use `action: "append"`** — never `upsert`. You are extending the anchor page, never rewriting it.
3. **Each edit's `path` must be the anchor page's path.** The `body` is what gets appended to the anchor.
4. **Each edit's `body` is exactly ONE paragraph** (1–3 sentences) that:
   - explains *why* the two concepts are related — not just that they are,
   - ends with a `[[wiki/concepts/<slug>.md]]` link pointing at the neighbor being referenced.
5. **Cross-link only to neighbors explicitly listed in the input below.** Do not invent links to pages not named here.
6. **The relation must be substantive.** The test: "Could I write a sentence a reader actually wants to read that explains how these two concepts sit next to each other?" If the only honest answer is "they share some vocabulary" or "they're both in the same field", SKIP that neighbor — it's a false positive.
7. **At most {{maxEdits}} edits total**, even if all neighbors are related. Pick the most substantive connections.
